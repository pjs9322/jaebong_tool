  - **작업 원칙:** 기존 `index.html`의 **모든 기능을 100% 동일하게 유지**한 채 파일 구조만 분할해야 함.

## 🥈 Phase 2: 핵심 버그 수정 및 데이터 설계 개선 (Critical Bugs & Data Flow)
기본적인 작업 흐름(Workflow)에서 데이터가 꼬이거나 유실되는 치명적인 문제들을 먼저 해결합니다.

- [ ] **사진 없는 요청사항 작성 지원 및 State 예외 처리 (Bug Fix)**
  - **이슈:** 이미지가 없는 상태에서 작업을 저장하거나 렌더링될 때(`S.img === null`) Canvas 참조 에러가 나거나, 이전 요청서의 이미지가 잔상처럼 남는 상태(State) 꼬임 현상 발생.
  - **작업 방향:** 
    - 이미지가 없어도 메모만으로 "이미지가 없는 요청사항" 카드로 정상 등록되도록 허용.
    - 이전 히스토리의 이미지 잔상이 남지 않도록 저장 및 렌더링 루프 진입 시 썸네일(`thumb`) 및 캔버스 초기화를 명시적으로 보장하여 데이터 무결성 개선.

- [ ] **작업 유실 방지 시스템 (Confirm Alert 다운타임 방어)**
  - **이슈:** 캔버스 위에 메모를 그리는 등 수정을 진행 중인 상태에서, '화면 캡처', '파일 열기', 'Ctrl+V 붙여넣기' 등을 수행하면 확인 절차 없이 기존 진행 상황이 증발함.
  - **작업 방향:**
    - 새 이미지를 불러오는 트리거 상단에 검증 로직 추가 (`S.state !== 'IDLE'` 또는 `S.annos.length > 0`).
    - `confirm("작업 중인 내용이 사라집니다. 계속하시겠습니까?")` 로직을 통해 덮어쓰기 전 사용자 동의 확보.

## 🥉 Phase 3: UX 경험 개선 및 사용 편의성 고도화 (UX/UI Polish)
사용자가 작업하면서 느낄 수 있는 불편함이나 행동 유도(Affordance)의 부재를 개선합니다.

- [ ] **튜토리얼 반복 노출 개선**
  - **이슈:** 새로고침할 때마다 우측 하단의 튜토리얼이 무조건 실행됨.
  - **작업 방향 (localStorage 기반):**
    - `localStorage.getItem('isTutorialDone')`이 `true`면 초기 스포트라이트 애니메이션 표출을 스킵하고, 진행을 완료하면 `setItem('isTutorialDone', 'true')` 처리. 백엔드 통신 없이 브라우저 단에서 영구 보관.

- [ ] **히스토리(Queue) 아이템 삭제 시 안전장치**
  - **이슈:** 목록에 추가된 작업을 지울 때 실수로 휴지통 버튼을 눌러도 복구/취소 기회가 없음.
  - **작업 방향:** `renderQueue()`의 배열 삭제 로직 진입 전에 `confirm("정말 이 항목을 삭제하시겠습니까?")` 경고 팝업 추가.

- [ ] **메모 저장 UX 명확화**
  - **이슈:** 캔버스 메모를 작성 중(`MEMO_WAIT` 상태)일 때 실수로 '목록에 추가하기'를 누르면 에러 Alert만 떠서 당황스러움.
  - **작업 방향 (시각적 피드백 제공):**
    - "메모의 [확인] 버튼을 먼저 눌러주세요" 안내와 함께, 우측 메모 패널의 **[확인] 버튼이 붉은색 빛으로 반짝거리는 CSS 애니메이션**을 1초간 발생시켜 사용자가 다음에 해야 할 행동을 직관적으로 유도.

- [ ] **PDF 내보내기 템플릿(Design) 일원화**
