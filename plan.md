# � 재봉툴 PRO v13 종합 서비스 개선 분석 및 로드맵 (Plan)

전체 코드베이스(HTML, CSS, JS 5종)의 구조와 로직을 리뷰하여, 현재 완성된 클라이언트 사이드 기능을 넘어 실제 상용화 및 사용자 경험(UX) 관점에서 도입이 필요한 **'서비스 측면 개선 방향 5가지'**를 분석했습니다.

---

## 🚀 1. 모바일 및 태블릿 환경 터치 이벤트 지원 (모바일 최적화)
### 🔍 현상 및 한계점
현재 핵심 기능인 `js/core/canvas.js` 내의 박스 그리기(Draft), 메모 선택, 리사이징 로직이 오직 마우스 이벤트(`onmousedown`, `onmousemove`, `onmouseup`)에만 의존하여 작성되어 있습니다.
이로 인해 사용자가 아이패드(태블릿)나 모바일 환경에서 손가락이나 펜슬을 이용해 화면을 탭하거나 드래그할 때 기능이 전혀 동작하지 않는 치명적 한계가 존재합니다.

### 🎯 개선 방향
- `canvas.js`의 이벤트 리스너에 `touchstart`, `touchmove`, `touchend`를 맵핑합니다.
- 터치 이벤트 발생 시 `e.touches[0].clientX` 등으로 좌표(`getPos`)를 계산하도록 분기를 추가하여, 현장의 태블릿/모바일 기기에서도 화면을 캡처하고 바로 메모를 남길 수 있는 **크로스 플랫폼 앱**으로 도약시킵니다.

## � 2. 예기치 않은 종료 대비 '로컬 데이터 자동 저장' (Session Recovery)
### 🔍 현상 및 한계점
현재 모든 의뢰 내역(`S.history`)과 작업 상태(`S.annos`)가 메모리 상에 휘발성으로 존재합니다.
사용자가 오랜 시간 수십 개의 수정 사항을 기록하다가, 실수로 브라우저 탭을 닫거나 새로고침(F5)을 누르면 그동안의 피땀 어린 의뢰 내역이 단 1초 만에 전부 증발해 버립니다.

### 🎯 개선 방향
- 사용자가 리스트(`history.js`)에 요소를 `push` 하거나 삭제할 때마다, 브라우저의 `localStorage` 또는 `IndexedDB`에 안전하게 `JSON.stringify(S.history)` 데이터를 자동 저장(Auto-save)합니다.
- `main.js`의 `startup()` 초기화 단계에서 저장된 로컬 데이터가 있는지 묻고, **"이전에 작업하던 의뢰서 내역이 있습니다. 이어서 작성하시겠습니까?"** 라며 복구(Recovery)해주는 시스템을 도입하여 고객 경험의 안정성을 획기적으로 상승시킵니다.

## 🚀 3. 실제 협업을 위한 '백엔드 연동과 진짜 공유 URL' 생성
### 🔍 현상 및 한계점
현재 우측 상단의 **[웹 링크 생성(공유)]** 버튼을 누르면 나타나는 뷰어 모드(`viewerLayer`)와 채팅 시스템은, 단순히 UI 껍데기만 뷰어 화면으로 덮어씌운 클라이언트 전용 목업(Mock-up) 상태입니다.
실제로는 카카오톡이나 이메일로 다른 작업자(에이전시, 개발자 등)에게 던져줄 수 있는 "실제 접속 가능한 웹 링크"가 만들어지지 않습니다.

### 🎯 개선 방향
- **BaaS(Backend as a Service)** 즉, Firebase Firestore나 Supabase를 프로젝트에 연동합니다.
- 공유 버튼 클릭 시, 클라이언트는 `S.history`의 Base64 이미지들과 JSON 데이터를 데이터베이스에 통째로 업로드하고 고유 해시 ID(예: `jaebong.com/view?id=aX9b2`)를 발급받아 클립보드에 복사해줍니다.
- 이 링크로 접속한 실제 작업자는 뷰어 모드로 진입하며, 내장된 채팅창은 setTimeout 가짜 답변이 아닌 웹소켓 실시간 채팅 기능으로 이어지게 확장합니다.

## 🚀 4. 고해상도(Retina) 디스플레이 번짐 현상 개선 및 이미지 편집 도구
### 🔍 현상 및 한계점
현재 `canvas.width`와 `height`를 불러온 이미지의 1:1 픽셀로만 세팅합니다. 맥북의 레티나(Retina) 디스플레이나 최신 스마트폰 등 고해상도(High-DPI) 모니터 환경에서 보면 선과 텍스트가 미세하게 흐리고 픽셀이 깨져 보일 수 있습니다. (PDF 출력 시 화질 저하의 원인)
더불어 사용자가 클립보드로 이미지를 복사해 넣었을 때 불필요한 주변 영역을 잘라내고(Crop) 싶어도 도구가 없습니다.

### 🎯 개선 방향
- **Device Pixel Ratio (DPR) 렌더링 도입:** 캔버스 1픽셀을 렌더링할 때 브라우저의 기기 픽셀 비율(`window.devicePixelRatio`)을 곱하여(scale) 캔버스를 2~3배 뻥튀기한 뒤 CSS로 사이즈를 맞추는 방식으로 그리기 화질을 극강으로 끌어올립니다.
- **간편 이미지 크롭(Crop) 모드 추가:** 캔버스 상단에 자르기 기능 토글을 두어 원하는 영역만 남기고 자체 크로핑할 수 있게 만들면 업로더의 작업 속도 효율이 상승합니다.

## 🚀 5. 실행 취소 (Undo / Redo) 기능 부재
### 🔍 현상 및 한계점
사용자가 박스를 신나게 치다가 한두 개를 실수하거나 위치를 되돌리고 싶을 때, 기존 시스템은 박스를 클릭하여 '삭제'하고 처음부터 각도를 잡아 다시 그려야 하는 일방통행 방식입니다.

### 🎯 개선 방향
- 드로잉(`S.annos`)의 스냅샷 배열을 구축합니다. `Ctrl + Z` 또는 `Cmd + Z` 키보드 단축키를 입력할 때마다 직전 배열 상태로 캔버스를 `render()` 해주는 스택 기반의 Undo/Redo 아키텍처를 도입하여 조작 편의성을 대폭 향상시킵니다.

### ✅ 종합의견
현재 재봉툴 프로젝트는 <b>Client-side 단일 어플리케이션</b>으로서 바닐라 JS(Vanilla JS)와 FSM 패턴을 훌륭하게 아울러 핵심 기능을 매우 밀도 있게 구현했습니다. 
위에서 제시한 "기기 호환(터치), 데이터 보존(로컬 저장소), 실시간 공유(백엔드 API)" 3대 과제만 다음 버전(v14~)에 마일스톤으로 세운다면, 시장에 실제 프로덕트로 내놓아 구독(SaaS) 형태의 서비스로 운영하기에 손색이 없는 막강한 퀄리티가 될 것입니다.
