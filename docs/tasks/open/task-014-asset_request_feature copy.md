# 테스크 플랜: 홈페이지 자산관련 요청 기능 (워크 모드 통합)

이 문서는 `jaebong` 프로젝트에 '홈페이지 자산관련' 요청 기능을 추가하기 위한 상세 구현 계획입니다. 기존의 이미지 캡처 중심 방식에서 벗어나, 캡처 없이도 자산(도메인, 서버 등) 관련 요청을 처리할 수 있는 전용 '워크 모드'를 구축합니다.

---

## 1. 개요 및 목적
유지보수 현장에서 빈번한 **도메인 갱신, 서버 이전, 계정 분실** 등은 이미지 캡처가 불필요하거나 불가능한 경우가 많습니다. 이를 위해 캔버스 영역을 **자산 선택 그리드**로 전환하는 전용 모드를 도입하여, 정형화된 데이터를 빠르고 정확하게 입력받습니다.

---

## 2. 주요 구성 요소 및 동작 방식

### A. 자산 워크 모드 (Asset Work Mode)
- **모드 전환**: 카테고리 바의 "🏠 홈페이지 자산관련" 버튼 클릭 시 진입.
- **캔버스 변환**: 기존 이미지 및 박스 그리기 기능이 비활성화되고, 캔버스 영역에 **토글형 체크박스 그리드**가 오버레이로 표시됩니다.
- **항목 선택**: 인프라, 이전, 계정 분실 등 카테고리별 아이템을 토글 방식으로 다중 선택.

### B. 모드 전환 정책 (Mode Switching Policy)
이미지 기반의 일반 모드와 자산 모드 간의 자유로운 전환을 지원하되, 데이터 정합성을 위해 다음 규칙을 적용합니다.

1. **자산 모드 진입 시 데이터 보호**:
   - 일반 모드에서 작성 중인 정보(캡처 이미지, 메모 박스 등)가 있을 경우, 소실 위험을 알리는 **Confirm 안내** 후 승인 시에만 전환됩니다.
   - 전환 즉시 기존의 캡처 이미지(`S.img`)와 메모 데이터(`S.annos`)는 필드에서 전면 제거(초기화)됩니다.
2. **일반 모드로의 복귀 및 정보 파기**:
   - 자산 모드에서 일반 카테고리(내용 수정, 기능 개발 등) 버튼을 눌러 복귀할 수 있습니다.
   - **중요**: 자산 모드에서 일반 모드로 전환하는 순간, 자산 모드에서 선택하거나 입력한 모든 정보(체크 항목, 상세 설명 등)는 **즉시 파기**됩니다.
   - 따라서 일반 모드로 복귀 버튼 클릭 시에도 소실 위험을 알리는 **Confirm 안내**가 필요합니다.
   - 복귀 후 새로운 캡처 이미지를 추가하면 완전히 새로운 이미지 기반 요청사항 작성 상태가 됩니다.

### C. UI/UX (Frontend)
- **자산 선택 그리드**: 캔버스 영역을 꽉 채우는 반응형 그리드 레이아웃.
- **메모 패널 통합**: 항목 선택 시 우측 메모 패널(`UI.memoPanel`)이 활성화되어 '구체적 상황, 긴급도, 관련 업체'를 입력받음 (기존 메모 인터페이스 재활용).
- **히스토리 카드**: 
  - 이미지 썸네일 대신 '자산 요청 전용 아이콘 및 요약 텍스트' 표시.
  - "캡처 없음" 문구와 함께 선택된 자산 항목들이 리스트업됨.

### C. 데이터 구조 (Data Schema)
`S.history`에 저장될 `type: 'asset'` 객체 정의:
```javascript
{
  id: number,
  category: 'asset',
  isAsset: true,
  assets: string[],      // ['domain', 'server', ...]
  desc: string,          // 상세 내용 및 업체 정보 합산
  urgency: 'high'|'mid'|'low',
  urgencyLabel: string,
  date: string,
  thumb: null,           // 이미지 없음
  full: null,
  baseImgSrc: null,
  annos: []              // 좌표 기반 메모는 사용하지 않음
}
```

## 3. 세부 구현 단계

### 1단계: 모드 전환 로직 및 UI 준비
- `state.js`에 `ASSET` 상태 추가 및 `canvas.js`의 이벤트 리스너(mousedown 등)에서 상태 분기 처리.
- `index.php`의 캔버스 영역 내부에 자산 선택 그리드(`#assetModeOverlay`) 마크업 추가.
- `main.css`에 자산 모드 전용 스타일(그리드, 체크박스 토글 등) 정의.

### 2단계: 자산 선택 및 데이터 입력 전산화
- 캔버스 오버레이에서의 항목 선택 로직 구현 (Set 객체 활용).
- 선택 시 우측 메모 패널을 강제 활성화하여 상세 정보를 입력받는 flow 구축.
- '리스트 추가하기' 클릭 시 `S.history`에 자산 타입 데이터를 생성하여 삽입.

### 3단계: 리스트 및 뷰어 렌더링 엔진 보완
- `history.js`의 `renderQueue`: `isAsset`일 경우 이미지 태그 대신 자산 정보 렌더링.
- `viewer-main.js`: 공유 페이지에서 자산 카드를 클릭했을 때 이미지 팝업 대신 상세 정보를 보여주는 레이어 노출.

---

## 4. 고려 사항 및 리스크
- **이미지 의존성 제거**: 기존 코드에서 `S.img`가 `null`일 때 발생할 수 있는 렌더링 오류 철저히 점검.
- **수정 로직**: 자산 요청 리스트의 '수정' 클릭 시 다시 자산 워크 모드로 진입하고 선택 상태를 복구하는 로직 필요.
- **가독성**: 텍스트 중심의 요청이 이미지 요청 사이에서 묻히지 않도록 시각적 강조(전용 아이콘, 배경색 등) 적용.

---

> [!IMPORTANT]
> 자산 모드에서는 기존의 **박스 그리기(캐치) 기능이 차단**되며, 오직 프리셋된 자산 항목 선택과 텍스트 입력만 가능합니다.
