# Task 012: 크롬 확장프로그램 기반 원클릭 화면 캡처 및 URL 연동 기획

## Goal
현재 브라우저 보안 정책(Same-Origin Policy)으로 인해 웹페이지(재봉툴) 내에서 다른 탭의 URL을 직접 얻는 것이 불가능한 한계를 극복합니다.
**"전용 크롬 확장프로그램(Chrome Extension)"**을 도입하여, 사용자가 타겟 웹사이트(본인 홈페이지)에서 버튼 클릭 한 번만으로 URL과 화면 스크린샷을 동시에 취득하고, 이를 재봉툴 워크 캔버스로 유기적으로 연동하는 완벽한 흐름을 기획합니다.

## Target User UX (컴맹 사용자를 위한 유기적 시나리오)
컴퓨터 지식이 전무한 사용자가 타겟이므로, 확장프로그램 '설치'라는 진입 장벽을 최소화하고 마치 하나의 웹 서비스처럼 물 흐르듯 이어지도록 설계합니다.

1. **설치 유도 (웹 ➔ 스토어):**
   - 사용자가 재봉툴 웹사이트에 접속 시 "가장 쉬운 클릭 한 번으로 내 홈페이지를 캡처해오세요!"라는 큰 배너와 함께 **[크롬 확장프로그램 설치하기]** 버튼을 제공합니다.
   - 클릭 시 크롬 웹스토어의 재봉툴 전용 페이지로 이동하여 즉시 설치를 유도합니다.
2. **원클릭 캡처 (타겟 사이트 ➔ 확장프로그램):**
   - 설치 완료 후, 사용자가 자신의 홈페이지(예: 수선집.com)를 띄운 상태에서 브라우저 우측 상단 퍼즐 옆의 **[재봉툴 아이콘]**을 클릭합니다.
   - 확장프로그램 팝업이 뜨고 **[이 화면을 재봉툴로 보내기]** 버튼을 누릅니다. (또는 아이콘 클릭 즉시 자동 실행 옵션도 가능)
3. **데이터 수집 및 자동 이동 (확장프로그램 ➔ 재봉툴):**
   - 확장프로그램이 뒷단에서 `chrome.tabs.query({active: true})` API로 현재 탭의 **URL을 즉시 복사**하고, `chrome.tabs.captureVisibleTab` API로 **고화질 스크린샷 1장**을 찍습니다.
   - 데이터를 확보한 즉시, 확장프로그램이 새 탭을 열어 재봉툴(`https://susunzip.com/jaebong` 또는 로컬호스트 등)로 사용자를 강제 이동시킵니다.
   - 이때 파라미터나 메시지 전달 릴레이(Message Passing)를 통해 아까 확보한 URL과 이미지 데이터를 재봉툴 캔버스에 자동으로 꽂아줍니다!

## Architecture (3단계 시스템 설계)

### Phase 1: Chrome Extension 개발 (프론트엔드 외곽)
- **`manifest.json`**: 확장프로그램의 이름, 아이콘, 필요한 권한(`activeTab`, `scripting`, `storage` 등)을 선언합니다. `activeTab` 권한을 쓰면 설치 시 무시무시한 경고 문구를 피할 수 있습니다.
- **`popup.js` / `popup.html`**: 확장프로그램 아이콘을 눌렀을 때 나타나는 작은 UI 창입니다.
- **`background.js` (Service Worker)**: 실제 캡처와 재봉툴 탭을 띄우는 코어 로직입니다. 선택된 화면을 캡처하고, 재봉툴 탭을 새로 띄운 뒤 `postMessage`나 `localStorage/IndexedDB`를 통해 데이터를 넘겨주는 역할을 합니다.

### Phase 2: Web App 수신부 개발 (재봉툴 메인)
- 확장프로그램이 열어준 재봉툴 웹 페이지(`js/main.js` 및 `imageLoader.js`)에는 수신기가 필요합니다.
- 페이지 진입 시 URL 파라미터(`?ext_capture=true`) 혹은 확장프로그램 전용 메시지 리스너(`window.addEventListener('message', ...)`)를 통해 **URL 텍스트**와 **Base64 이미지 데이터**가 무사히 도착했는지 감지합니다.
- 데이터가 확인되면 튜토리얼이나 초기 화면을 모두 생략하고, 즉시 **STEP 02 캔버스 위에 이미지를 깔고 Page URL 입력란에 주소를 텍스트로 박아**줍니다.

### Phase 3: Seamless 설치 연동 (선택사항)
- `Inline Installation` 기술(현재 크롬 정책상 외부 웹스토어 이동 필수이나 팝업 형태 고려)을 활용하거나, 친절한 15초짜리 움짤(GIF) 가이드를 제공하여 '설치 후 아이콘 고정(Pin)' 과정을 시각적으로 매우 친절하게 안내해야 합니다.

## Result
크롬 확장프로그램을 자체 개발하여 크롬 웹스토어에 심사/등록(1~3일 소요) 해야 하는 공수가 듭니다만, "사용자가 직접 URL을 복사 붙여넣기 한다"는 행동 자체를 완벽하게 근절할 수 있는 궁극적이고 유일한 해결책입니다.
