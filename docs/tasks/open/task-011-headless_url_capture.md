# Task 011: URL 입력 기반 탭 이탈 없는(Tabless) 백그라운드 웹페이지 캡처 기획

## Goal
현재 사용자가 수동으로 브라우저 탭을 이동하여 캡처 대상을 '공유 선택' 해야 하는(`getDisplayMedia` API 사용) 과정의 불편함을 해소하고자 합니다. 
재봉툴 탭을 이탈하지 않고 **URL만 입력하면 백그라운드(서버 단)에서 해당 웹페이지를 알아서 전체 화면으로 캡처하여 도화지로 가져오는 매끄러운 캡처 경험**을 구축합니다.

## Background (기술적 한계와 극복 방안)
- **현재 구현 방식의 한계:** 클라이언트(JS) 브라우저 보안 정책상, 별도 설치 프로그램이나 확장 플러그인(Chrome Extension) 없이는 현재 탭이 아닌 '외부 탭/화면'을 마음대로 캡처할 수 없습니다. 
- **해결책 (Server-Side Rendering):** 클라이언트 안에서 조작하는 것이 불가능하므로, 백엔드 서버에서 가상의 브라우저(Headless Browser)를 띄워 타겟 URL에 접속한 뒤 스크린샷을 찍고 그 결과물(이미지 URL)만 재봉툴로 넘겨받는 아키텍처 도입이 필수적입니다.

## Plan (2가지 구현 전략 옵션)

### 🚀 Option A: 상용 스크린샷 API(Third-party) 연동 (추천 - MVP 단계)
- **개념:** `Microlink API`, `ScreenshotAPI.net`, `ApiFlash` 등 전문가들이 미리 만들어둔 이미지 생성 클라우드 서비스를 활용합니다.
- **작동 흐름:** 재봉툴 내에 URL 주소 입력란 제공 ➝ "캡처해오기" 클릭 ➝ 프론트엔드가 즉시 써드파티 API로 요청 ➝ 로딩 스피너 (수 초 대기) ➝ 전체 화면 이미지 도화지 세팅.
- **장점:** 자체 백엔드 인프라(현재 카페24 호스팅 등)를 수정하거나 복잡하게 건드릴 필요 없이 **JS 코드 몇 줄만으로 1~2시간 내외로 구현이 가능**해 MVP 시점에서 효용이 무척 높습니다.
- **단점/한계:** 무료 제공 티어(보통 월 100~1000회 제한)가 존재하며, 트래픽이 커질 경우 유료 전환 비용이 발생합니다. 회원가입/로그인 팝업이 뜨는 사이트는 막힌 화면이 캡처될 수 있습니다.

### 🛠️ Option B: 자체 Headless 크롬 캡처 서버 구축 (장기적 방향)
- **개념:** Node.js, `Puppeteer` 또는 `Playwright` 라이브러리를 사용해 독자적인 렌더링 서버를 구축합니다.
- **작동 흐름:** 클라이언트 ➝ 우리 자체 서버의 `/api/capture` 엔드포인트 호출 ➝ 가상 브라우저가 해당 사이트 접속 및 스크롤 다운(지연 로딩 이미지 확보) ➝ 스크린샷 저장 후 반환.
- **장점:** 캡처 직전에 눈에 거슬리는 팝업창이나 배너를 스크립트로 강제 종료시켜버리는 등 완벽한 비스포크 커스텀(Bespoke Customization) 제어가 가능하고 이용 횟수 제한이 없습니다.
- **단점/한계:** 단순 웹 호스팅 수준에서는 크롬이 돌아가지 않으므로 AWS EC2, Lambda 또는 Vercel 같은 클라우드 환경 재세팅이 필요하며, 장기적인 서버 유지비용과 높은 개발 공수가 듭니다.

## UX/UI 개선 시나리오 (프론트 디자인)
1. **버튼 변경 및 모달 팝업:**
   기존 `[📸 내 홈페이지 화면 캡처]` 버튼을 누르면 탭 공유 다이얼로그가 뜨는 대신, 재봉툴 화면 중앙에 예쁜 **[URL 입력 모달]** 창이 표시됩니다.
2. **로딩 인디케이터:**
   사용자가 "https://..." 주소를 넣고 확인을 누르면 화면 전역에 `"AI 어시스턴트가 웹사이트를 캡처해오고 있습니다! 🚀"` 텍스트와 함께 움직이는 로딩 모션이 표기됩니다.
3. **배치 완료:**
   탭 전환(깜빡임) 전혀 없이 스무스하게 캡처된 거대한 이미지가 작업 캔버스에 안착하게 됩니다.
