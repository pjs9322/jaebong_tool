# Task 005: 비로그인 다중 문서 관리 및 내 작업 목록(대시보드) 구축

## Goal
MVP 단계(Task 001)에서 적용된 '단일 문서 강제 리다이렉트' 제한을 해제하고, 로컬 디바이스 식별키(UUID)를 기반으로 사용자가 과거에 작성했던 **모든 문서(의뢰서)의 목록을 확인하고 관리할 수 있는 [내 작업 목록(대시보드)] UI**를 구축합니다.

## Background (요구사항 배경)
- Task 001 단일 문서 아키텍처의 DB 뼈대는 본 안건을 위해 이미 `owner_uuid`를 활용한 1:N 구조로 완벽히 준비되어 있음.
- MVP 이후, 사용자가 여러 건의 의뢰나 각기 다른 디자인을 저장해 두고 싶어 하는 니즈(Pain-point)를 해소해야 함.
- 마이그레이션(기존 백엔드 구조 변경)은 전혀 필요 없으며, 프론트엔드의 화면(라우팅/뷰) 추가만으로 달성 가능함.

## Plan
1. **API 앤드포인트 신설 (PHP/MariaDB):**
   - `GET /api/docs/list.php`: 헤더(또는 파라미터)로 전달된 `UUID`를 받아, 테이블 내 `owner_uuid`가 일치하는 모든 문서 배열(ID, 썸네일, 최근 수정일시 등)을 내림차순(최신순) 반환.
   
2. **클라이언트 화면(UI) 개발:**
   - 상단 헤더에 `[ 📝 내 작업 목록 ]` 버튼 신설 및 클릭 시 좌측에서 튀어나오는 사이드 투폴드(Sidebar) 혹은 전체 화면 오버레이 방식의 대시보드 모달 구현.
   - 각 리스트 타일 클릭 시 해당 문서 URL(`/?id=DOC-XXX`)로 라우팅되어 에디터 모드 즉시 재진입 유지.
   - 리스트 상단 내지 툴바에 무조건 새 도화지를 여는 `[ ➕ 새 문서 만들기 ]` 버튼 추가 (Task 001의 기존 자동 리다이렉트 분기문 제외 처리).

3. **용량 및 가비지(GC) 정책 연결 UI:**
   - 목록 화면에서 휴지통 아이콘 등을 통해 낱개 문서를 유저 스스로 완전 삭제(DB Delete)할 수 있는 컨트롤 로직 제공.
   - 비로그인 환경임을 명시하며 "최대 저장 갯수(Ex: 10개) 초과 시 오래된 문서는 임의로 파기될 수 있음" 같은 클라이언트단 안내(Alert) 검토 (선택 사항).
