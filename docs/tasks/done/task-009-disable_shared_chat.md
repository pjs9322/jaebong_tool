# Task 009: 공유 링크(Viewer 내) 채팅 영역 및 상태 변경 드롭다운 비활성화

## Goal
공유 링크를 통해 접속한 뷰어(Viewer) 모드 우측에 표시되는 '프로젝트 소통(채팅/히스토리)' 영역과 상태 변경 드롭다운(검토요청, 검토중, 검토완료) 요소를 임시로 비활성화하고, 의뢰 내용 확인에 더 집중할 수 있도록 레이아웃을 조정합니다.

## Background
- 현재 `index.html` 내에 구현되어 있는 `.view-sidebar` 영역(채팅/댓글 기능)과 상태 변경 등은 실시간 양방향 통신(WebSocket 등) 백엔드가 완벽하게 연동되지 않은 MVP 단계의 목업(Mockup) 성격이 강합니다.
- 채팅 및 상태 변경 기능이 작동하지 않으면 사용자에게 오해를 불러일으킬 수 있으므로, 당분간 비활성화하고 본연의 '요청사항 열람' 기능에 집중합니다.

## Plan
1. **HTML 수정:**
   - `index.html` 내의 `<div class="view-sidebar"> ... </div>` 영역을 주석 처리하거나, `display: none` 속성을 인라인 혹은 CSS 클래스로 추가합니다.
   - 공유 링크 내 상태 변경 드롭다운(검토요청, 검토중, 검토완료) 요소도 주석 처리 혹은 `display: none` 속성으로 보이지 않게 처리합니다.
   
2. **CSS 레이아웃 조정:**
   - 우측 사이드바가 숨겨짐에 따라 여백이 발생할 수 있습니다. `css/viewer.css` 또는 `css/main.css`에서 중앙 `.view-main` 혹은 `.view-paper`의 너비(width)나 `margin: 0 auto;` 정렬 방식 등을 확인하여, 화면 중앙에 보기 좋게 배치될 수 있도록 맞춤 개선합니다.

3. **JS 이벤트 정리 (선택):**
   - 만약 JS 코드(`js/core/viewer.js` 혹은 `js/main.js`) 내에 채팅 버튼(`id="btnSendChat"`) 클릭 이벤트 리스너나 채팅 리스트 업데이트 방면 혹은 상태 드롭다운 변경 관련 로직이 있다면, 오류가 나지 않도록 주석 처리합니다.
