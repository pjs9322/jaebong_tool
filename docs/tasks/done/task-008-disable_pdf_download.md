# Task 008: PDF 다운로드 버튼 및 기능 임시 비활성화

## Goal
현재 제공되는 'PDF 다운로드' 버튼과 연관된 클라이언트 기반(`html2canvas`, `jsPDF`) PDF 생성 기능을 임시로 비활성화하고, 사용자 경험(UX)을 '웹 URL 공유' 중심으로 유도합니다.

## Background (요구사항 배경 및 의도 분석)
전체 프로젝트(Task 001 ~ Task 007) 맥락을 분석한 결과, PDF 기능을 임시 비활성화하는 데에는 다음과 같은 기술적/전략적 의도가 내포되어 있습니다.

1. **클라우드 생태계(웹 뷰어) 중심의 Lock-in 및 바이럴 유도**
   - 현재 실시간 공유 및 다중 문서 관리(Task 005) 기능이 도입되어 문서는 철저히 '클라우드 기반'으로 관리되고 있습니다. 정적인 PDF 파일을 다운로드하여 플랫폼 외부로 이탈하게 두는 것보다, 고유 공유 링크(`/?id=DOC-XXX`)를 통해 타인에게 전달하게 함으로써 서비스 유입 트래픽을 늘리고 작업자를 플랫폼 내에 체류토록 유도하는 것이 서비스 성장에 훨씬 유리합니다.
   
2. **프론트엔드 PDF 렌더링의 기술적 한계 및 기능 파편화 문제 예방**
   - 이전 업데이트(Task-000)를 보면 `html2canvas`와 `jsPDF`를 이용한 렌더링에서 '디자인 파편화' 현상이 보고된 바 있습니다. 특히 최근 진행 중인 고해상도 이미지 크롭(Task 007) 및 모바일 터치 지원(Task 006) 과제가 병합되면, 클라이언트 브라우저 단에서 무거운 고해상도 캔버스를 조작하고 PDF로 구워내는 작업은 심각한 기기 성능 저하, 브라우저 크래시, 화질 열화를 유발할 위험이 매우 높습니다.

3. **핵심 가치 집중 및 유지보수 리소스 최적화**
   - 개발 리소스가 제한된 상황에서 브라우저별로 파편화되는 완벽한 PDF 출력을 위해 지속적으로 뷰어 CSS를 동기화하고 버그를 잡는 것은 비효율적입니다. '웹 링크 실시간 공유'라는 훨씬 우수하고 안정적인 대안이 성립된 상태이므로, 불안정한 PDF 출력 기능은 당분간 비활성화하고 코어 기능(데이터 연동, 대시보드)의 안정성을 높이는 데 집중하는 것이 합리적인 전략입니다.

## Plan
1. **UI 요소 비활성화:**
   - `index.html` 내부에 존재하는 `<button class="btn-black" id="exportBtn">PDF 다운로드</button>` 요소를 DOM에서 주석 처리하거나 `display: none` 적용.
   - 불필요한 네트워크 리소스 낭비를 막기 위해 `index.html`의 `jspdf` CDN 스크립트 태그 렌더링 주석 처리.
   
2. **이벤트 및 모듈 분리:**
   - `js/main.js` 내의 `initPdfExport()` 호출 단위를 주석 처리하여 불필요한 이벤트 리스너 마운트 방지.
   - `js/services/pdfExport.js` 모듈 자체는 추후 서버사이드(SSR) 방식의 PDF 렌더링이나 개선된 기능으로 재도입될 수 있으므로 파일 원본은 삭제하지 않고 보존.

3. **사용자 경험(UX) 보완:**
   - PDF 버튼을 찾지 못한 유저가 당황하지 않도록 기존 '공유하기(링크 복사)' 버튼의 시인성을 높이거나 툴팁 등을 통해 'URL 공유' 기능을 적극적으로 안내 검토.
