# Task 010: 구글 애널리틱스(GA4) 연동 및 이벤트 트래킹 구축

## Goal
사용자의 접속 통계 및 주요 기능(문서 생성, 공유, 이미지 캡처 등) 사용 현황을 파악하기 위해 구글 애널리틱스(Google Analytics 4) 추적 코드를 서비스 전체에 연동합니다.

## Background
- 서비스 런칭(MVP) 초기 단계에서 유저들이 어느 경로로 유입되고 어떠한 기능을 주로 사용하는지 정량적인 데이터 검증(Data Tracking)이 필수적입니다.
- 특히 우리가 집중하고 있는 "웹 링크를 통한 바이럴(공유) 전환율" 등을 측정하기 위해서는 단순한 페이지뷰(PV) 뿐만 아니라 특정 버튼 클릭에 대한 커스텀 이벤트 측정이 필요합니다.

## Plan
1. **기본 추적 코드 삽입:**
   - `index.html` 파일의 `<head>` 태그 최상단에 발급받은 Google 태그(gtag.js) 스크립트를 삽입합니다. (Tracking ID는 개발/관리자와 협의하여 세팅)

2. **주요 커스텀 이벤트(Event Tracking) 심기:**
   - `js/main.js` 또는 `js/util/api.js` 등 기능이 실행되는 접점에서 다음과 같은 주요 사용자 행동을 트래킹하도록 코드를 추가합니다.
     - `document_create`: 새 문서(의뢰서)가 처음 막 생성되었을 때.
     - `image_import`: '내 홈페이지 화면 캡처' 혹은 '파일 열기' 등을 통해 캔버스에 이미지를 불러왔을 때.
     - `request_added`: 새 요청사항(메모/박스 등)을 작성하여 리스트에 추가(`saveReqBtn` 클릭)했을 때.
     - `link_shared`: '웹 링크 생성(공유)' 버튼을 눌렀을 때 (바이럴 핵심 지표).

3. **비로그인 유저 식별 연결:**
   - (선택) Task 001에서 구현한 익명 `UUID` 값을 GA의 `user_id` 속성으로 넘겨주어, 기기 기반의 정교한 퍼널(Funnel) 분석이 가능하도록 고도화합니다.
