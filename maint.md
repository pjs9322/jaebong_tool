# 재봉툴(Jaebong) 유지보수 수칙

본 문서는 재봉툴 프로젝트의 코드 품질과 유지보수성을 유지하기 위한 핵심 수칙을 정의합니다.

## 1. 스타일링 원칙 (CSS)
- **인라인 스타일 지양**: 모든 HTML 엘리먼트에서 인라인 스타일(`style="..."`) 사용을 점진적으로 줄여 나갑니다.
- **점진적 리팩토링**: 새로운 기능을 추가하거나 기존 기능을 수정할 때, 해당 영역의 인라인 스타일을 CSS 클래스로 옮기는 작업을 병행합니다. 특히 CSS 파일과 인라인 스타일이 중복되는 경우, 인라인 스타일을 제거하고 CSS 파일로 통합하는 것을 원칙으로 합니다.
- **클래스 기반 스타일링**: 모든 스타일은 전용 CSS 파일(`css/main.css`, `css/viewer.css` 등)에서 클래스 또는 ID 선택자를 통해 정의하는 것을 지향합니다.
- **!important 지양**: 스타일 우선권 해결을 위해 `!important`를 사용하는 대신, CSS 선택자의 구체성(Specificity)을 높이거나 구조를 개선하여 해결합니다.
- **디자인 시스템 준수**: `:root`에 정의된 변수(`--action-pink`, `--primary` 등)를 적극적으로 활용합니다.

## 2. 자바스크립트 (JS)
- **Style 객체 조작 최소화**: JS에서 `element.style.fontSize = '20px'`와 같이 스타일을 직접 조작하는 것을 지양합니다. 가급적 클래스를 토글(`classList.add/remove`)하는 방식을 사용합니다.
- **전역 변수 자제**: 전역 상태는 `store/state.js`의 `S` 오브젝트를 통해 중앙 집중식으로 관리합니다.

## 3. HTML/PHP
- **의미론적 마크업**: `div` 위주의 구성보다는 시맨틱 태그를 적절히 활용합니다.
- **불필요한 공백 제거**: 코드 가독성을 해치는 불필요한 공백과 줄바꿈을 정리합니다.
